                                                        I have some issues with the claim that think that 'm foreach f' can be translated to:

                                                        m flatMap {x => unit(f(x));()}

                                                        I think that in order to be correctly typed, the function argument for flatMap should have a result which is of the same monad type as m. unit(f(x)) meets this requirement, but () does not. So I think it should be:

                                                        m flatMap {x => unit(f(x))} ; ()

                                                        Should it not?

                                                        From the scala type hierarchy () which is of type scala.Unit, is not a subtype of all types, so () is not a subtype of the type class of m.
